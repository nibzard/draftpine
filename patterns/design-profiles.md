# Draftpine Design Profiles

Choose one profile before composing a screen. Profiles keep agents from
inventing a new visual system on every page while preserving Draftpine's
plain-HTML, Pico, Alpine, no-build contract.

## Editorial

Best for product marketing, comparison pages, feature pages, and narrative
content.

- Dense first viewport with proof visible below the hero.
- Medium-weight headings, tight section rhythm, compact proof panels.
- Cards only for repeated items; sections stay unframed.
- Use one accent for active proof, selected tabs, and primary CTAs.

## SaaS

Best for dashboards, billing, admin, settings, CRM, and operations screens.

- Quiet, utilitarian, scan-first layout.
- Tables, metric strips, filters, segmented controls, and sidebars are favored
  over large decorative heroes.
- Stable dimensions for toolbars, tabs, cards, and counters.
- Empty/error/success states are part of the workflow, not afterthoughts.

## Marketing

Best for homepages, launch pages, landing pages, and campaign pages.

- First viewport must communicate product/category, primary action, proof, and
  a hint of the next section.
- Use real UI artifacts, code, metrics, screenshots, or workflow states as
  proof. Avoid quiet placeholder boxes.
- Hero copy should be literal, not inflated.

## Docs

Best for documentation, quickstarts, changelogs, and guides.

- Navigation, table of contents, code panels, and callouts are the primary
  structure.
- Use short sections and scannable headings.
- Include copy buttons or install commands only when they visibly affect state.

## App

Best for multi-step workflows, onboarding, account areas, and internal tools.

- App shell first: nav, current page, task context, primary action.
- Preserve repeated-action ergonomics: clear state, obvious next step, compact
  controls.
- Use dialogs for focused actions and preserve accessible labels.
