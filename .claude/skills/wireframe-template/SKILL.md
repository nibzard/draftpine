Choose a Draftpine pattern recipe before editing files.

Process:

1. Read `AGENTS.md`.
2. Read `patterns/README.md`.
3. Open only the pattern files relevant to the requested screen.
4. Compare the user's prompt or screen packet to pattern groups, sections, states, and interactions.
5. Pick 5-9 patterns and explain the recipe in one or two sentences.
6. If starting a new screen, build the root `index.html`, `styles.css`, `app.js`, and `draftpine.config.json` from that recipe.

Reference examples:

- Billing, invoices, usage, plans: `examples/billing`.
- Metrics, reporting, status, overview: `examples/dashboard`.
- Deals, sales, stages, pipeline: `examples/crm-pipeline`.
- Setup, activation, checklist: `examples/onboarding`.

Examples are references, not required starting points. Do not force-fit a screen into an example.
